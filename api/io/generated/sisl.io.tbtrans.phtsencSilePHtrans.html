iter(group=True, dimension=True, variable=True, levels=-1, root=None)ÔÉÅ

Iterator on all groups, variables and dimensions.

This iterator iterates through all groups, variables and dimensions in the Dataset

The generator sequence will _always_ be:

  1. Group

  2. Dimensions in group

  3. Variables in group

As the dimensions are generated before the variables it is possible to copy groups, dimensions, and then variables such that one always ensures correct dependencies in the generation of a new SileCDF.

Parameters

  • group (bool (True)) ‚Äì whether the iterator yields Group instances

  • dimension (bool (True)) ‚Äì whether the iterator yields Dimension instances

  • variable (bool (True)) ‚Äì whether the iterator yields Variable instances

  • levels (int (-1)) ‚Äì number of levels to traverse, with respect to root variable, i.e. number of sub-groups this iterator will return.

  • root (str (None)) ‚Äì the base root to start iterating from.

Examples

Script for looping and checking each instance.

>>> for gv in self.iter():
...     if self.isGroup(gv):
...         # is group
...     elif self.isDimension(gv):
...         # is dimension
...     elif self.isVariable(gv):
...         # is variable

pivot(elec=None, in_device=False, sort=False)ÔÉÅ

Return the pivoting indices for a specific electrode (in the device region) or the device

Parameters

  • elec (str or int) ‚Äì the corresponding electrode to return the pivoting indices from

  • in_device (bool, optional) ‚Äì If True the pivoting table will be translated to the device region orbitals. If sort is also true, this would correspond to the orbitals directly translated to the geometry self.geometry.sub(self.a_dev).

  • sort (bool, optional) ‚Äì Whether the returned indices are sorted. Mostly useful if you want to handle the device in a non-pivoted order.

Examples

>>> se = tbtncSileTBtrans(...)
>>> se.pivot()
[3, 4, 6, 5, 2]
>>> se.pivot(sort=True)
[2, 3, 4, 5, 6]
>>> se.pivot(0)
[2, 3]
>>> se.pivot(0, in_device=True)
[4, 0]
>>> se.pivot(0, in_device=True, sort=True)
[0, 1]
>>> se.pivot(0, sort=True)
[2, 3]

See also

pivot_down

for the pivot table for electrodes down-folding regions

scattering_matrix(elec, E, k=0, sort=False)ÔÉÅ

Return the scattering matrix from the electrode elec

The scattering matrix is calculated as:

\[\Gamma(E) = i [\Sigma(E) - \Sigma^\dagger(E)]\]
Parameters

  • elec (str or int) ‚Äì the corresponding electrode to return the scattering matrix from

  • E (float or int) ‚Äì energy to retrieve the scattering matrix at, if a floating point the closest energy value will be found and returned, if an integer it will correspond to the exact index

  • k (array_like or int) ‚Äì k-point to retrieve, if an integer it is the k-index in the file

  • sort (bool, optional) ‚Äì if True the returned scattering matrix will be sorted according to the order of the orbitals in the non-pivoted geometry, otherwise the scattering matrix will be returned according to the pivoted orbitals in the device region.
